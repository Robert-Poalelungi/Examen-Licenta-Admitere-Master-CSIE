Q1 POO / C++

Q2 POO / C++
Ce este o functie membra?
Raspuns
O functie definita in interiorul unei clase; apartine acelei clase.
Q3 POO / C++
Ce este o functie statica?
Raspuns
O functie statica este o functie care poate fi accesata direct prin intermediul clasei; este asociata cu clasa in sine si nu cu o instanta a acesteia; folosirea unei functii statice nu este dependenta de initializarea unui obiect. Se acceseaza cu Clasa::metoda_statica().
Q4 POO / C++
Ce este un atribut static?
Raspuns
Un atribut care poate fi modificat fara a fi necesara initializarea unei instante a clasei; un atribut static nu apartine unei instante a clasei, ci a clasei in sine. Metodele accessor pentru atributele statice trebuie sa fie si ele statice. Se acceseaza cu Clasa::atribut_static.
Q5 POO / C++
Ce este un atribut constant?
Raspuns
Un atribut constant este un atribut a carui valoare nu poate fi modificata dupa initializarea sa.
Q6 POO / C++
Cand se poate initializa un atribut constant?
Raspuns
Doar in constructor, prin lista de initializare.
Q7 POO / C++
Ce este o functie virtuala?
Raspuns
O functie virtuala este o metoda a unei clase (parinte / mostenita) care poate fi suprascrisa in clasele derivate din aceasta.
Q8 POO / C++
Care sunt tipurile de constructori in cadrul unei clase si ce rol are fiecare?
Raspuns
Constructor fara parametri (default) – are rolul de a initializa o instanta a clasei cu valori predefinite pentru atributele sale. Constructor cu parametri – are rolul de a initializa o instanta a clasei cu valori custom pentru un set de atribute ale clasei (pot fi toate, poate fi unul singur). Constructor de copiere – are rolul de a initializa o instanta a clasei folosind valori dintr-o alta instanta a clasei care a fost initializata deja.
Q9 POO / C++
Care este diferenta intre rolul operatorului = si cel al constructorului de copiere?
Raspuns
Operatorul egal atribuie unei instante DEJA INITIALIZATE valori de la o alta clasa deja initializata. Constructorul de copiere initializeaza o instanta NOU CREATA folosind datele dintr-o instanta deja initializata.
Q10 POO / C++
Cand este apelat constructorul de copiere?
Raspuns
Cand instantiem un nou obiect pe baza unuia existent. Este apelat automat si: - in momentul in care trimitem un obiect prin valoare (nu referinta) ca parametru intr-o metoda - cand returnam un obiect prin valoare (nu referinta) dintr-o metoda
Q11 POO / C++
Cand este apelat operatorul=?
Raspuns
Cand atribuim unui obiect deja creat valorile altui obiect deja creat. La autoatribuire (a=a) sau atribuire in lant (a = b = c). In momentul in care atribuim unei instante deja create un obiect returnat prin valoare de o metoda.
Q12 POO / C++
Ce este un memory leak?
Raspuns
Apare in momentul in care nu dezalocam o zona de memorie alocata dinamic (heap). Variabilele din stack se dezaloca automat, cele din heap (new/malloc) trebuie dezalocate manual (delete/free).
Q13 POO / C++
Ce este un dangling pointer?
Raspuns
Un dangling pointer arata spre o zona de memorie ce a fost dezalocata / stearsa.
Q14 POO / C++
Cand se apeleaza destructorul?
Raspuns
Cand se iese din scope-ul in care a fost initializat obiectul.
Q15 POO / C++
Ce este memoria heap?
Raspuns
O zona de memorie destinata alocarii dinamice (in care putem retine pointeri) si care poate fi alocata manual (new/delete in C++, malloc/free in C).
Q16 POO / C++
Cum se genereaza un memory leak?
Raspuns
In momentul in care nu dezalocam o bucata din memoria heap pe care am alocat-o dinamic.
Q17 POO / C++

Raspuns
Permit accesul controlat la atributele private ale clasei. Sunt de doua tipuri: getter (get) – consultare, setter (set) – modificare. Sunt definite in zona public.
Q18 POO / C++

Raspuns
Functiile friend primesc acces la atributele private ale clasei prietene. Caracteristici: Nu primesc pointer-ul this, nu pot fi mostenite, nu sunt membre ale clasei.

Q27 POO / C++

Q29 POO / C++

Q30 POO / C++

Q42 Baze de date

Q43 Baze de date
Cate tipuri de normalizari exista?
Raspuns
FN1: Orice atribut este atomic (nu mai poate fi divizat). FN2: Este in FN1 + orice coloana non-cheie depinde de intreaga cheie primara (nu partial). FN3: Este in FN2 + nu exista dependente tranzitive. FN Boyce-Codd (FN 3.5): Dependentele functionale netriviale contin in stanga o cheie candidata. FN4: Este in FNBC + nu se manifesta mai mult de o dependenta multi-valoare. FN5: Este in FN4 + fiecare dependenta jonctiune e implicata de o cheie candidata. Exista si FN6 (foarte rar folosit).

Q60 SQL
Ce sunt intersectia si diviziunea in SQL?
Raspuns
Intersectia: operatie pe doua relatii R1 si R2; rezulta R3 cu schema identica continand tuplurile comune. Sintaxa SQL: SELECT ... INTERSECT SELECT ... Diviziunea: operatie pe o relatie R(A,B) si un set de tupluri r(B); rezulta o relatie cu atributele din A care au corespondent pentru TOATE valorile din B ale lui r.

Q63 Baze de date
Ce inseamna ACID?
Raspuns
Atomicitate – o tranzactie are loc fie complet, fie deloc. Consistenta – BD este valida atat inainte cat si dupa tranzactie; daca nu respecta restrictiile de integritate, tranzactia este respinsa. Izolare – doua tranzactii simultane nu interfereaza una cu cealalta. Durabilitate – odata finalizata, modificarile raman in BD chiar daca apar defectiuni ale sistemului.

Q73 SQL

Q85 POO / C++
Ce sunt supraincarcarea functiilor si supradefinirea metodelor?
Raspuns
Supraincarcarea functiilor: cel putin doua functii au acelasi nume, dar difera prin numarul si tipul parametrilor. Identificarea se face la compilare (early binding). Polimorfism slab. Supradefinirea: functiile au aceeasi semnatura dar implementari diferite in clase derivate. Alegerea functiei se face la executie (late binding). Polimorfism puternic. Necesita marcarea cu virtual.
Q86 POO / C++
Ce este supraincarcarea operatorilor? Cum se realizeaza?
Raspuns
Supraincarcarea operatorilor permite specializarea operatorilor existenti pentru tipuri de date definite de utilizator. Se realizeaza prin functii cu denumirea: operator Tehnici: - Metoda membra: cand primul operand este de tipul clasei (transpus in this) - Functie globala: cand primul operand nu este de tipul clasei (ex: <<, >>) Tipuri de operatori: comparatie (==, !=, <, >), aritmetici (+, -, *, /), acces ([], (), ->), atribuire (=), citire/afisare (<<, >>), unari (++, --, !), binari.
